# Golden Example 2 — Cut for time without breaking sense

## Input
After the mistake pass, a clip is still ~75s. Mid-clip there is a 12s tangent about an unrelated anecdote;
near the end the creator calls back to it ("…and that's why the story I told you earlier matters").

## A great output asserts
- [ ] The tangent is cut with reason-class `time` (distinct from `mistake`/`filler` — the report shows the difference).
- [ ] The coherence gate catches the later callback line: it is also cut or trimmed so no orphaned reference survives.
- [ ] The time ladder ran in order (redundancy → tangent → weakest support) — core-message content was never cut for time.
- [ ] Final runtime ≤60s; if that had been impossible coherently, the report would say so explicitly instead of delivering a broken 60s cut.
- [ ] Kept transcript reads start-to-finish as one complete message for a first-time viewer.

## A bad output
Hits exactly 60s but the video references content that no longer exists; or protects the tangent and cuts a core point instead.
